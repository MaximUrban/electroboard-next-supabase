# ElectroBoard MVP — Next.js + Supabase

Простой MVP редактора схем для электрики:
- план помещения как `JPG / PNG / PDF`
- фигуры: прямоугольник, окружность, линия
- электрические элементы: розетка, выключатель
- кабельные линии
- калибровка масштаба по 2 точкам
- смета кабеля по группам и типам
- сохранение проектов в Supabase
- хранение файлов плана в Supabase Storage

---

## 1. Что куда заливать

### В Supabase
Нужно создать:
1. **проект**
2. **таблицу и политики** через SQL
3. **bucket `plans`** для файлов плана  
   Это создаётся SQL-скриптом `supabase/schema.sql`

### В Vercel
Нужно залить **весь этот проект целиком**:
- папка `app`
- папка `components`
- папка `lib`
- `package.json`
- `tsconfig.json`
- `next.config.mjs`
- `.env.local` у себя локально, а на Vercel те же значения добавить в Environment Variables

## 2. Совсем по шагам

### Шаг 1. Создайте проект в Supabase
1. Зайдите в Supabase
2. Нажмите **New project**
3. Придумайте имя, пароль и регион
4. Дождитесь запуска проекта

### Шаг 2. Возьмите ключи Supabase
В Supabase откройте:
**Project Settings → API**

Скопируйте:
- `Project URL`
- `anon public key`

### Шаг 3. Выполните SQL
В Supabase откройте:
**SQL Editor → New query**

Вставьте туда содержимое файла:

`supabase/schema.sql`

Нажмите **Run**.

Это создаст:
- таблицу `projects`
- bucket `plans`
- простые MVP-политики доступа

### Шаг 4. Создайте локальный `.env.local`
Скопируйте `.env.example` в `.env.local` и заполните:

```env
NEXT_PUBLIC_SUPABASE_URL=ваш Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш anon public key
NEXT_PUBLIC_SUPABASE_BUCKET=plans
```

### Шаг 5. Проверьте локально
Откройте терминал в папке проекта:

```bash
npm install
npm run dev
```

Откройте:
`http://localhost:3000`

### Шаг 6. Загрузите код в GitHub
Создайте новый репозиторий и загрузите туда **весь проект**.

### Шаг 7. Залейте в Vercel
1. Зайдите в Vercel
2. Нажмите **Add New → Project**
3. Выберите ваш GitHub-репозиторий
4. Нажмите **Deploy**

### Шаг 8. Добавьте переменные окружения в Vercel
В Vercel откройте:
**Project Settings → Environment Variables**

Добавьте:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `NEXT_PUBLIC_SUPABASE_BUCKET`

Потом нажмите **Redeploy**.

## 3. Что делает приложение

### План помещения
- загрузка JPG / PNG / PDF
- для PDF в фоне показывается **первая страница**
- исходный файл можно хранить в Supabase Storage

### Инструменты
- `Выбор`
- `Прямоугольник`
- `Окружность`
- `Линия`
- `Кабель`
- `Розетка`
- `Выключатель`
- `Калибровка`

### Масштаб
1. Выбираете инструмент **Калибровка**
2. Ставите 2 точки
3. Вводите реальный размер в метрах
4. Нажимаете **Применить калибровку**

После этого длина кабеля считается в метрах.

### Кабельная смета
Считаются только объекты типа **Кабель**:
- по группе: `A1`, `P1` и т.д.
- по типу кабеля: например `ВВГнг-LS 3x2.5`

### Привязка устройств
Розетки и выключатели:
- получают группу
- автоматически привязываются к ближайшей кабельной линии той же группы

### Это MVP

## 7. Если что-то не работает

### Ошибка сохранения в Supabase
Проверьте:
- SQL точно выполнен
- bucket `plans` создался
- переменные окружения правильные

### План не виден
- `JPG/PNG` должны открываться сразу
- `PDF` рендерится как картинка первой страницы

### После деплоя Vercel пустой экран
Почти всегда причина одна из двух:
- не добавлены environment variables
- не нажали redeploy после добавления переменных
